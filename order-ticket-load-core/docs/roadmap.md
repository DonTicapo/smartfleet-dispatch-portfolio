# Roadmap — Order / Ticket / Load Core

## Milestones
1. M1: canonical schema + migration set
2. M2: event-backed order/ticket/load service
3. M3: domain APIs with auth and audit logging
4. M4: integration contracts for dispatch, telematics, plant edge, customer portal

## v1 success criteria
- bounded context is clear
- contract surfaces are documented
- one narrow vertical slice works end-to-end
- no sibling responsibility leakage

## Defer until after v1
- advanced optimization
- broad UI polish
- cross-project analytics inside this repo
- speculative features not required for the first slice
